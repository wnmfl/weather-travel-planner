# 算法设计文档

## 1. 天气友好度评分算法

### 评分维度

- **温度因素** (权重 30%): 15-25°C 最舒适
- **降雨因素** (权重 35%): 降雨概率越低越好
- **风力因素** (权重 15%): 风力等级 0-2 级最佳
- **紫外线因素** (权重 10%): 指数 0-5 最好
- **湿度因素** (权重 10%): 40-60% 最舒适

### 总分计算

总分 = 100 - （温度惩罚 + 降雨惩罚 + 风力惩罚 + UV惩罚 + 湿度惩罚） 总分 = max（0， min（100， 总分））


### 评分等级

- 90-100: ⭐⭐⭐⭐⭐ 极好
- 75-89: ⭐⭐⭐⭐ 很好
- 60-74: ⭐⭐⭐ 良好
- 45-59: ⭐⭐ 一般
- 30-44: ⭐ 较差
- 0-29: ❌ 很差

## 2. 路线规划算法

### 问题定义

输入: N 个景点及其地理位置、游玩时间、天气数据
输出: 优化的景点访问顺序

### 算法步骤

1. **景点适宜度评分**: 景点评分 × 天气友好度
2. **贪心初始化**: 从评分最高的景点开始选择
3. **2-opt 优化**: 尝试交换相邻景点改进路线
4. **时间分配**: 根据天气调整每个景点的游玩时间

### 时间复杂度

- 贪心初始化: O(n²)
- 2-opt 优化: O(n² × iterations)
